# JT-022 - Chat History Schema Migration

## What Changed

JT-022 adds a new Alembic migration for compact persisted chat history storage.
The migration creates the local SQLite `chat_messages` table with `conversation_id`, `role`, `content`, compact `citations_json`, compact `tool_outputs_json`, and `created_at` columns.
It also adds a conversation/time index for efficient `GET /chat/history` reads when the Phase 5 chat API is implemented.

## Why

FR-5.6 requires chat history to be persisted locally in SQLite with compact citations and tool outputs.
This schema keeps chat history in the same local database as the rest of the app while preserving the `applications` and `application_events` source-of-truth invariant.
The citation and tool-output payloads remain compact JSON arrays instead of new relational source-of-truth tables.

## Constraints

The table stores no secrets, OAuth tokens, API keys, telemetry, or outbound email state.
The `role` column is constrained to `user`, `assistant`, `tool`, or `system`.
The compact JSON payload columns are required to contain valid JSON arrays.
