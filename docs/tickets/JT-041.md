# JT-041 - Add Chat Page Shell

## Summary

JT-041 owns the empty chat route shell for Phase 0.
The `/chat` route renders `frontend/src/pages/Chat.tsx` as a disabled conversation surface that reserves the page entry point without starting backend chat, streaming, retrieval, provider calls, or persisted chat history.

## Product Mapping

This work supports the Phase 0 groundwork deliverable for an empty `/chat` route shell.
It keeps Phase 5 RAG chat behavior out of scope while preserving the future path for FR-5 chat capabilities.
It does not change local storage, database schema, provider behavior, LLM behavior, telemetry, or secrets handling.

## User-Facing Behavior

Users can open `/chat` from the primary navigation.
The page explains that chat arrives in Phase 5.
The message composer is disabled until the backend chat endpoint, history store, grounding checks, and agent behavior exist.
The copy states that future quantitative chat answers must use constrained tools rather than raw SQL from an LLM.

## Scope Boundaries

JT-041 does not implement streamed responses.
JT-041 does not implement chat history.
JT-041 does not implement semantic retrieval.
JT-041 does not add backend chat routes or agent graph behavior.
JT-041 does not call an LLM or expose private email content.

## Verification

Relevant frontend verification is `npm run test -- App.test.tsx` from `frontend/` for the route shell tests.
Full frontend verification is `npm run check` from `frontend/`.
