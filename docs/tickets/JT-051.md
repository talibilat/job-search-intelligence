# JT-051 - Add Google OAuth Setup Guide

## What changed

JT-051 is implemented by `docs/google-oauth-setup.md`.
The guide documents the user-created Google OAuth client path for Gmail ingestion.
It covers Google Cloud project setup, OAuth consent configuration, Desktop OAuth client creation, local storage of the downloaded client JSON, the `gmail.readonly` scope, and the app's `GET /auth/gmail` and callback expectations.

## Why it changed

JobTracker is local-first and must not ship shared Google credentials.
The user needs clear setup instructions for bringing their own Google OAuth client while keeping OAuth token material behind `SecretStore` and preserving the v1 read-only Gmail boundary.

## Verification

- `uv run pytest tests/test_ticket_docs.py -q`
- `uv run pytest tests/test_setup_wizard_copy.py tests/test_ticket_docs.py -q`

## Usage notes

- Follow `docs/google-oauth-setup.md` before starting Gmail authorization.
- Store the downloaded Google OAuth client JSON outside the repository.
- Keep `JOBTRACKER_GMAIL_SCOPES` set to `https://www.googleapis.com/auth/gmail.readonly`.
- Do not place OAuth client JSON contents, client secrets, access tokens, refresh tokens, or secret-store files in committed files.
