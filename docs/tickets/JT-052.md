# JT-052 - Add LLM Provider Setup Guide

## What changed

JT-052 is implemented by `docs/llm-provider-setup.md`.
The guide documents the setup values and privacy boundaries for Azure OpenAI and Ollama as the first supported LLM provider paths.
It covers provider choice, `classification_mode`, non-secret environment values, `SecretStore` API-key storage, local-only Ollama behavior, provider health checks, and readiness checks.

## Why it changed

JobTracker needs a first-run setup path that can configure hosted Azure OpenAI or local Ollama without committing or logging provider credentials.
The setup guidance preserves the local-first model and keeps deterministic dashboard counts separate from LLM-generated narrative or classification work.

## Usage notes

- Use Azure OpenAI with `JOBTRACKER_CLASSIFICATION_MODE=hybrid` for the hosted cost-controlled path.
- Use Ollama with `JOBTRACKER_CLASSIFICATION_MODE=local` for the local-only path.
- Store Azure OpenAI API keys through `SecretStore`, not in `backend/.env`.
- Keep Ollama base URLs local so prompt content is not sent to an unintended remote endpoint.
