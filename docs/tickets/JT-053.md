# JT-053 - Add Setup Wizard Copy

## What changed

JT-053 is implemented by the typed setup wizard copy in `frontend/src/setupWizardCopy.ts` and the setup page rendering in `frontend/src/pages/SetupPage.tsx`.
The copy covers provider selection, classification mode, Gmail read-only authorization, and privacy boundaries.
The backend smoke test `backend/tests/test_setup_wizard_copy.py` pins the required copy terms.

## Why it changed

The first-run setup shell must make provider, mode, Gmail, and privacy decisions explicit before later tickets add durable setup persistence and credential writes.
The copy keeps Azure OpenAI, Ollama, `hybrid`, `llm`, `local`, `gmail.readonly`, user-owned OAuth clients, `SecretStore`, no shared credentials, no telemetry, and deterministic dashboard counts visible to the user.

## Verification

- `uv run pytest tests/test_ticket_docs_jt_053.py -q`
- `uv run pytest tests/test_setup_wizard_copy.py tests/test_ticket_docs_jt_053.py -q`

## Usage notes

- The setup page shell renders the copy without storing secrets in the browser.
- OAuth, provider adapters, setup submission, credential storage, and durable backend setup behavior remain separate tickets.
- The UI copy must not imply that dashboard counts come from an LLM.
