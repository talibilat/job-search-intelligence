# JT-054 - Add Minimal Playwright Smoke Harness

## What changed

JT-054 is implemented by `frontend/playwright.config.ts`, `frontend/tests/smoke/phase0-shell.pw.ts`, and the `npm run test:smoke` script in `frontend/package.json`.
The smoke test starts the Vite shell, stubs sync API responses, verifies the Phase 0 overview shell, exercises the manual sync UI, checks the dashboard empty state, and navigates to the setup page.

## Why it changed

The project needs a small browser-level smoke harness for critical Phase 0 user paths without introducing a broad end-to-end suite.
This protects setup, sync status, and dashboard shell behavior while keeping verification fast and local-first.

## Verification

- `npm run test:smoke`
- `uv run pytest tests/test_ticket_docs_jt_054.py -q`

## Usage notes

- Run `npm install` in `frontend/` before invoking the smoke test in a fresh worktree.
- If Chromium is missing on a machine, install it with `npx playwright install chromium` before running `npm run test:smoke`.
- The smoke test intentionally stubs sync responses and does not require a configured Gmail account.
