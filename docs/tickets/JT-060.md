# JT-060 - Gmail Metadata Normalization

## What Changed

Gmail metadata listing now normalizes provider metadata before it crosses the `EmailProvider` boundary.
Opaque Gmail message IDs, thread IDs, and history cursors are trimmed without changing their case.
Sender, recipient, and CC email addresses are parsed with display names trimmed and addresses lowercased.
Duplicate recipients are removed while preserving first-seen order.
Gmail system labels are trimmed, canonicalized to uppercase, deduplicated, and blank labels are dropped.
Custom label IDs keep their original casing after trimming.
Header values such as subject and RFC 822 message ID are trimmed.
Parsed message timestamps are normalized to timezone-aware UTC values.

## Why

JT-060 supports Phase 1 Gmail ingestion for `FR-1`, `FR-6`, `NFR-1`, `NFR-4`, and `NFR-7`.
Normalized metadata gives later filtering, classification, aggregation, and reconciliation stages stable identifiers and timestamps without retaining body-derived snippets.
The change stays local-first and keeps Gmail-specific behavior behind the email provider adapter.

## How To Verify

Run the Gmail metadata listing tests:

```sh
uv run --project backend pytest backend/tests/test_gmail_message_listing.py
```

Run the broader related backend tests:

```sh

```
