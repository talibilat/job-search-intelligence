# JT-063 - HTML-To-Text Normalization

## What Changed

JT-063 added retained email body normalization at the provider DTO boundary.
HTML MIME bodies are converted into normalized plain text before retention, non-content HTML elements such as scripts and styles are stripped, visible anchor text is preserved without retaining link URLs, and retained body DTOs reject raw HTML fields.

Plain-text retained bodies that still look like raw HTML now fail validation instead of being silently retained.
This keeps downstream raw email storage free of raw HTML by default.

## Why It Changed

Phase 1 ingestion needs retained body text for selected job-search messages, but retaining raw HTML would increase privacy exposure and complicate downstream classification, embedding, and citation behavior.
This supports `FR-1`, `FR-6`, `NFR-1`, `NFR-4`, `NFR-7`, and the project privacy rule that raw HTML is not retained by default.

## Usage Notes

Use `EmailBodySource.HTML_CONVERTED` when a provider returns an HTML MIME body that should be retained as normalized plain text.
Use `EmailBodySource.TEXT_PLAIN` only for already-plain text bodies.
Do not pass `raw_html` or other raw HTML retention fields through retained body DTOs.
