# JT-064 - Raw Email Repository Writes

JT-064 implements idempotent repository writes for raw email records in Phase 1 ingestion.

The raw email repository now supports retained and debugging body writes even when the body is fetched before a metadata row exists locally.
In that case, the repository creates a valid `raw_emails` row with the provider message ID, provider, thread ID, retained body text, retention state, empty labels, and the body fetch timestamp as the ingestion timestamp.

When metadata later arrives for the same provider message ID, the metadata-only replay updates metadata fields such as sender, recipient, subject, sent timestamp, and labels without downgrading the existing `retained` or `debugging` body text.

The existing metadata-only write path remains idempotent and still preserves previously retained bodies.
The retained-body write path is duplicate-safe, so replaying the same retained or debugging body does not create duplicate rows.
