# JT-074 - Sync API Routes

## What changed

`POST /sync` now resolves the persisted Gmail account from the local SQLite `email_connections` table before starting manual sync.
`GET /sync/status` continues to report the current or last manual sync state through the existing `EmailSyncStatus` DTO.
The sync API still returns the typed not-configured `400` response when no Gmail account has been persisted.

## Why

FR-1.5 requires a manual sync trigger and a sync status endpoint.
The route shell already existed, but the default connection dependency still returned no connection.
This ticket wires the route to persisted non-secret Gmail connection metadata so the runtime can pass the account and `SecretRef` to the Gmail provider without exposing OAuth token material.

## How to verify

Run `uv run --project backend pytest backend/tests/test_sync_api.py -q`.
For the full backend gate, run `uv run --project backend ruff check .`, `uv run --project backend mypy .`, and `uv run --project backend pytest`.
