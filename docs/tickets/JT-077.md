# JT-077 - Add Sync Now UI Action

## What Changed

The overview page now shows the current manual sync status from `GET /sync/status`.
It includes a `Sync now` button that calls `POST /sync` through the generated frontend API client.
The UI renders the returned sync state, sync mode, message count, and raw email count after a manual run.
Typed API errors from the sync endpoint are shown in the page instead of being hidden.

## Why

FR-1.5 requires a manual `Sync now` trigger from the local web app.
This keeps Gmail ingestion user-triggered and local-first while using the backend API added by JT-074.
The UI does not access Gmail, OAuth tokens, provider cursors, page tokens, or email bodies directly.

## How To Use

Start the local backend and frontend.
Open the overview page at `/`.
Use the `Sync now` button in the sync readiness section.
If Gmail is not connected, the page shows the typed backend error telling the user what is missing.
After a successful run, the page displays the latest sync result returned by the backend.

## Verification

Run `npm run test -- src/App.test.tsx` from `frontend/` to verify the overview sync action behavior.
Run `npm run check` from `frontend/` before merging to verify the generated API client, typecheck, lint, tests, and build.
