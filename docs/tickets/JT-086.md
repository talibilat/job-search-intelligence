# JT-086 - Azure OpenAI Provider

JT-086 adds an Azure OpenAI chat-completions adapter behind the provider-neutral `LLMProvider` protocol.
This supports Phase 2 classification and extraction work without letting provider-specific request payloads leak into pipeline code.

## What Changed

- Added `backend/app/providers/llm/azure_openai.py` as the Azure-specific adapter file.
- The adapter builds Azure chat-completion requests from `LLMGenerationRequest` DTOs.
- API keys are read from the configured `SecretStore` using the existing `llm_api_key` ref for `azure_openai/api_key`.
- The adapter maps Azure responses back to `LLMGenerationResponse`, including finish reasons and token usage when Azure returns usage data.
- Azure transport failures are converted to typed `LLMProvider` errors with public-safe messages.

## How To Use

Configure non-secret Azure metadata through the existing settings:

```text
JOBTRACKER_LLM_PROVIDER=azure_openai
JOBTRACKER_CLASSIFICATION_MODE=hybrid
JOBTRACKER_AZURE_OPENAI_ENDPOINT=https://<resource>.openai.azure.com
JOBTRACKER_AZURE_OPENAI_API_VERSION=2024-06-01
JOBTRACKER_AZURE_OPENAI_CHAT_DEPLOYMENT=<chat-deployment>
JOBTRACKER_AZURE_OPENAI_EMBEDDING_DEPLOYMENT=<embedding-deployment>
```

Store the API key through the configured secret-store backend, not in `.env`.
The adapter reads this ref:

```text
kind=llm_api_key
provider=azure_openai
name=api_key
```

Callers use `AzureOpenAIProvider.generate(...)` with an `LLMGenerationRequest`.
If `request.model` is set, the adapter treats it as the Azure chat deployment override for that call.
Otherwise it uses `JOBTRACKER_AZURE_OPENAI_CHAT_DEPLOYMENT`.

## Verification

The focused adapter tests cover:

- Protocol conformance.
- SecretStore-backed API key lookup.
- Azure chat-completions URL and payload construction.
- JSON-object response format mapping.
- Request model deployment override.
- Response parsing into provider-neutral DTOs.
- Missing API key handling.
- Malformed response handling.
- Transient Azure transport error mapping.

Run the focused tests with:

```sh
uv run --project backend pytest backend/tests/test_azure_openai_provider.py
```
