# JT-087 Ollama Provider

## Scope

JT-087 implements local Ollama chat generation behind the backend `LLMProvider` protocol.
It maps to Phase 2, FR-2, NFR-1, NFR-2, NFR-4, and NFR-5.

## What Changed

The backend now has `app.providers.llm.ollama.OllamaLLMProvider`.
The adapter accepts provider-neutral `LLMGenerationRequest` values and returns provider-neutral `LLMGenerationResponse` values.
It posts to Ollama's local `/api/chat` endpoint using the configured non-secret `ollama_base_url`, `ollama_chat_model`, `llm_timeout_seconds`, and `llm_max_retries` settings.
It rejects non-local Ollama base URLs so local mode does not silently send prompt content to a remote host.
The provider registry also rejects non-local Ollama base URLs during provider configuration validation.
The backend dependency layer can resolve the selected Ollama settings into an `LLMProvider` instance for downstream services and routes.
The dependency maps invalid selected Ollama configuration into the standard typed API error shape instead of leaking provider exceptions through FastAPI.
The urllib transport builds a proxy-disabled opener so local Ollama calls are not routed through environment or system HTTP proxies.
It maps text generation to normal Ollama chat calls and JSON-object generation to Ollama's `format: "json"` request field.
It maps provider token counts into `LLMTokenUsage` when Ollama returns them.
It maps transport failures, timeouts, unavailable local runtime responses, and malformed provider payloads to the existing public-safe LLM provider errors.

## Privacy And Security

Ollama requires no API key in this adapter.
The adapter does not read `.env` directly and does not log prompts, completions, provider payloads, or private email content.
The local provider path keeps classification and extraction calls on the user's machine when `JOBTRACKER_LLM_PROVIDER=ollama` and `JOBTRACKER_CLASSIFICATION_MODE=local` are selected.

## How To Use

Start Ollama locally and pull the configured chat model.

```sh
ollama pull llama3.1
```

Use these non-secret settings through the setup flow or environment overrides.

```env
JOBTRACKER_LLM_PROVIDER=ollama
JOBTRACKER_CLASSIFICATION_MODE=local
JOBTRACKER_OLLAMA_BASE_URL=http://127.0.0.1:11434
JOBTRACKER_OLLAMA_CHAT_MODEL=llama3.1
JOBTRACKER_LLM_TIMEOUT_SECONDS=60
JOBTRACKER_LLM_MAX_RETRIES=2
```

Application code should instantiate `OllamaLLMProvider` at the provider-selection boundary and then depend only on `LLMProvider`.
