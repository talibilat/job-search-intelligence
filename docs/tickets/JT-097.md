# JT-097 - Pre-Run Classification Estimate

## What Changed

JT-097 adds a `GET /classification/estimate` API endpoint for checking bulk classification scope before running it.
The endpoint returns the number of retained candidate emails that need classification for the current model and prompt version, estimated prompt tokens, estimated completion tokens, total estimated tokens, and an estimated USD cost when pricing rates are configured.

## Why

This supports FR-0.4 and NFR-2 by making classification cost visible before a bulk run starts.
The estimate is local and deterministic, so it does not call an LLM and does not expose email content.

## How It Works

The candidate count comes from local SQLite `raw_emails` rows with non-empty retained candidate body text.
Rows are included when they have no `email_classifications` row, when their stored classification uses a different model, or when their stored classification uses a different prompt version from `JOBTRACKER_CLASSIFICATION_PROMPT_VERSION`.
Metadata-only emails, debugging-only retained bodies, and empty or whitespace-only retained bodies are excluded because they are not bulk classification candidates.
If the configured local database does not exist yet, the read-only estimate reports zero candidates instead of creating a new database file.

Token estimates use a configurable heuristic of `ceil(body_text_chars / chars_per_unit)` plus per-candidate prompt overhead and completion budget.
Local classification mode reports zero cost.
Non-local modes compute cost only when input and output pricing rates are configured.

## Configuration

The estimate uses these settings.

- `JOBTRACKER_CLASSIFICATION_ESTIMATE_CHARS_PER_UNIT`
- `JOBTRACKER_CLASSIFICATION_ESTIMATE_PROMPT_OVERHEAD_UNITS`
- `JOBTRACKER_CLASSIFICATION_ESTIMATE_COMPLETION_UNITS_PER_CANDIDATE`
- `JOBTRACKER_CLASSIFICATION_INPUT_COST_PER_1K_UNITS_USD`
- `JOBTRACKER_CLASSIFICATION_OUTPUT_COST_PER_1K_UNITS_USD`

Leave the pricing settings at `0` when provider pricing is unknown.
In that case non-local estimates still report candidate and token counts, but cost is marked unavailable.

## Verification

Focused tests cover candidate filtering, stale prompt-version inclusion, stale model inclusion, token estimation, configured non-local pricing, local zero-cost behavior, missing-database behavior, and the new settings in `backend/.env.example`.
