# JT-099 - Golden Set Fixture File

## What changed

Added `backend/evals/golden_set.jsonl` as a private-data-free JSONL fixture scaffold for Phase 2 classification evaluation.
The file contains 30 synthetic email cases with expected job-vs-not labels and classification categories.
Added `backend/tests/test_golden_set_fixture.py` to validate the JSONL schema, private-data attestation, synthetic sender domains, unique case IDs, negative examples, and core positive category coverage.

## Why

This maps to `FR-2.7` and the Phase 2 requirement for a golden-set classification eval.
The fixture gives the future JT-100 eval runner stable, non-private records to score precision and recall without copying inbox data into the repository.
This ticket does not implement `backend/evals/run_eval.py` or enforce the 90 percent precision and 85 percent recall gate because JT-100 and JT-101 own those steps.

## How to use

Each line in `backend/evals/golden_set.jsonl` is one complete JSON object.
Each object has `schema_version`, `case_id`, `contains_private_data`, `email`, `expected`, and `rationale` fields.
The `expected.is_job_related` field is the job-vs-not label that the eval runner should use for precision and recall.
The `expected.category` field preserves the planned category label for category-level checks when classifier behavior exists.

## How to verify

Run `uv run --project backend pytest backend/tests/test_golden_set_fixture.py`.
For the full backend gate, run `uv run --project backend ruff check .`, `uv run --project backend mypy .`, and `uv run --project backend pytest`.
