# JT-103 - Role Normalization

## What Changed

Added `normalize_role_title()` in `backend/app/services/normalization.py`.
The helper converts extracted role titles into deterministic grouping keys for Phase 2 aggregation.

## Why

Aggregation depends on matching application evidence across emails that may describe the same role with different casing, punctuation, seniority labels, title levels, abbreviations, or work-location notes.
This keeps the future application grouping key stable without changing the `applications` schema or adding an API surface.

## Behavior

The helper returns `None` for missing or punctuation-only titles.
It lowercases, ASCII-folds, removes punctuation noise, collapses whitespace, expands common role abbreviations such as `SWE`, `SDE`, `SRE`, and `ML`, removes seniority labels, level tokens, common location tokens, and work-arrangement phrases, and canonicalizes common software, data, machine-learning, product, and project role families.
It preserves meaningful descriptors such as `back end`, `front end`, `growth`, and `machine learning` so later grouping does not collapse distinct role families.
