# JT-120 - Phase 2 Pipeline Smoke Test

## What Changed

Added a backend smoke test that drives the Phase 2 path across heuristic filtering, structured LLM classification and extraction, and aggregation.

The smoke test uses the private-data-free `basic_job_search.json` synthetic fixture as raw email evidence, adds one synthetic metadata-only non-candidate email, evaluates broad job-search filter decisions for every email, feeds retained candidates through a deterministic fake LLM provider, and aggregates the accepted extraction results into one application timeline.

It also reruns aggregation on the same accepted results and asserts the `applications` and `application_events` row counts stay stable.

## Why

JT-120 covers the Phase 2 integration seam for FR-2, NFR-2, NFR-4, and NFR-5.

The test protects the product invariant that factual job-search answers come from one clean `applications` table and its event timeline.

It keeps the check local and deterministic by using synthetic fixture data and a fake LLM provider instead of live email or provider calls.

## Verification

Run focused backend verification from the repository root:

```sh
uv run --project backend pytest backend/tests/test_phase2_pipeline_smoke.py -q
uv run --project backend ruff check backend
```

Run mypy from `backend/` because its configured targets are relative to that directory:

```sh
uv run mypy
```

Shared README documentation records this smoke test in the current backend capability and verification inventory.
