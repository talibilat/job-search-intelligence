# JT-121 - Metrics Service Foundation

## What Changed

Added typed foundational metric DTOs and a backend `MetricsService`.

The service computes deterministic foundational counts from the canonical `applications` table through the existing `ApplicationRepository`.

It currently returns total applications, distinct companies, fixed-order status counts, and a generation timestamp.

Added a FastAPI dependency seam so later `/metrics/*` endpoint tickets can inject the service without putting metric logic in route handlers.

## Why

JT-121 starts Phase 3 dashboard work by centralizing metric logic outside API routes.

The service keeps dashboard facts deterministic and tied to the `applications` table instead of any LLM output.

## Verification
