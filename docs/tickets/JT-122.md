# JT-122 - Metrics Repository Queries

## What Changed

Expanded `MetricsRepository` with deterministic query methods for Phase 3 metrics.

The repository now exposes total application counts, rate metrics, funnel stages, application-volume timeseries, and breakdown rows for planned dashboard dimensions.

Added typed Pydantic DTOs for rate, funnel, timeseries, and breakdown query results.

Added focused repository tests that seed a local SQLite database and verify the metric math against synthetic applications and event timelines.

## Why

JT-122 prepares the repository layer for the later `/metrics/rates`, `/metrics/funnel`, `/metrics/timeseries`, and `/metrics/breakdown` endpoints.

The queries keep dashboard metric math deterministic and tied to the canonical `applications` table plus `application_events` timeline.

## Verification

```sh
uv run --project backend pytest backend/tests/test_metrics_repository_queries.py -q
```
