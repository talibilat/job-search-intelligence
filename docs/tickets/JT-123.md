# JT-123 - Metrics Filter DTOs

## What Changed

Added a typed `MetricsFilter` Pydantic DTO for dashboard metric filters.

The DTO covers status, source, sponsorship, first-seen date range, role, salary band, and work mode.

It normalizes timezone-aware dates to UTC, trims role text, rejects blank roles, and rejects inverted salary bands.

Added focused DTO tests for accepted filter fields and validation errors.

## Why

JT-123 creates the typed boundary that later metrics endpoints can use to keep filter handling consistent across dashboard views.

The DTO keeps metric filters deterministic and aligned with the existing `applications` table fields.

## Verification

Run focused backend verification from the repository root:

```sh
uv run --project backend pytest backend/tests/test_metrics_filter_dtos.py -q
```
