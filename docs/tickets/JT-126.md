# JT-126 - Metrics Funnel Endpoint

## What Changed

Added deterministic `GET /metrics/funnel` for Q-16.
The endpoint returns the approved funnel stages in order: `applied`, `screen`, `interview`, `final`, and `offer`.
It accepts the shared dashboard filter query fields so funnel numbers compose with URL-backed dashboard filters.

## Why It Changed

Q-16 asks for the full application funnel.
Dashboard funnel counts must reconcile with local SQLite `applications` and `application_events` rows and must not be produced by an LLM.

## Funnel Contract

`applied` counts filtered applications.
`screen` counts filtered applications with any response-like evidence.
`interview` counts filtered applications with an `interview_scheduled` event.
`final` is explicitly `0` until final-round evidence is represented in the stored event model.
`offer` counts filtered applications with an `offer` event after an `interview_scheduled` event in deterministic event order.

## Usage Notes

Request `/metrics/funnel` for unfiltered funnel counts.
Add dashboard filter query fields such as `source=linkedin` or `status=interview` to return funnel counts for the same filtered population shown elsewhere on the dashboard.
