# JT-127 - Metrics Timeseries Endpoint

## What Changed

Added `GET /metrics/timeseries`.

The endpoint returns a typed `MetricsTimeseriesResponse` with daily application-volume points from `MetricsRepository`.

Added a `MetricsTimeseriesService` and FastAPI dependency so the route stays thin.

Added focused API and OpenAPI tests for the timeseries endpoint.

## Why

JT-127 exposes deterministic Phase 3 application-volume trend data without involving any LLM path.

The endpoint reads from the canonical `applications.first_seen_at` values through `MetricsRepository`.

## Verification

Run focused backend verification from the repository root:

```sh
uv run --project backend pytest backend/tests/test_metrics_timeseries_api.py -q
uv run --project backend ruff check backend
```

Run mypy from `backend/` because its configured targets are relative to that directory:

```sh
uv run mypy
```

Run frontend contract verification from `frontend/`:

```sh
npm run check
```
