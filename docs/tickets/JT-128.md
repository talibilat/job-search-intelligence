# JT-128 - Metrics Breakdown Endpoint

## What Changed

Added `GET /metrics/breakdown` with a required `dimension` query parameter.

The endpoint returns a typed `MetricsBreakdownResponse` containing deterministic breakdown rows from `MetricsRepository`.

Added a `MetricsBreakdownService` and FastAPI dependency so the route stays thin.

Added focused API and OpenAPI tests for source breakdown rows.

## Why

JT-128 exposes deterministic Phase 3 segmentation data without involving any LLM path.

The endpoint reads from the canonical `applications` table and `application_events` timeline through `MetricsRepository`.

## Verification

Run focused backend verification from the repository root:

```sh
uv run --project backend pytest backend/tests/test_metrics_breakdown_api.py -q
uv run --project backend ruff check backend
```

Run mypy from `backend/` because its configured targets are relative to that directory:

```sh
uv run mypy
```

Run frontend contract verification from `frontend/`:

```sh
npm run check
```
