# JT-129 - Deterministic Metric Tests

## What Changed

Added focused backend coverage that loads the checked-in `basic_job_search.json` synthetic fixture through `SyntheticFixtureRepository`.

The test verifies `MetricsRepository` outputs for total applications, rejections, rates, funnel stages, application timeseries, and every currently implemented breakdown dimension.

## Why

JT-129 requires metric math to be verified against private-data-free synthetic fixtures.

This keeps Phase 3 dashboard metrics tied to the canonical local SQLite `applications` table and `application_events` timeline.

## Verification

Executed from the repository root:

```sh
uv run --project backend pytest backend/tests/test_metrics_repository_queries.py -q
uv run --project backend ruff check backend/tests/test_metrics_repository_queries.py
```

Both commands passed on 2026-07-08.

Executed from `backend/` because the configured mypy targets are relative to that directory:

```sh
uv run mypy
```

The command passed on 2026-07-08 with no issues in 210 source files.

## Usage Notes

The test uses only private-data-free fixture content with `contains_private_data` set to `false`.

It does not call an LLM and does not require live email or provider credentials.

Time-to-response and time-to-rejection coverage are not included here because those metric interfaces are not present in this branch's merged dependency surface yet.
