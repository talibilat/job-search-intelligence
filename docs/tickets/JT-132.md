# JT-132 - Rates And Funnel Widgets

## What Changed

The dashboard now renders the Q-16 application funnel from `GET /metrics/funnel`.
The existing deterministic rate cards remain in the metrics overview for response, rejection, ghost, application-to-interview, and interview-to-offer rates.
The rate cards and funnel widget reload with the active dashboard filters so they stay aligned with the filtered application table and breakdown widgets.

## Why It Changed

JT-132 requires conversion, rejection, ghost, and funnel views on the deterministic dashboard.
Rates were already rendered from `GET /metrics/rates`; this ticket adds the missing funnel surface after JT-126 introduced the backend funnel endpoint and extends the rates endpoint to accept the shared dashboard filter query fields.

## Verification

Run focused frontend verification from `frontend/`:

```sh
npm run test -- DashboardPage
```

Run focused backend verification from the repository root:

```sh
uv run --project backend pytest backend/tests/test_metrics_rates_api.py -q
```

Run full frontend verification from `frontend/`:

```sh
npm run check
```

## Usage Notes

Open `/dashboard` and review the Application funnel section.
Apply dashboard filters such as status or role to reload the rates and funnel from `/metrics/rates` and `/metrics/funnel` with the same query fields.
The `final` stage is shown as zero until final-round evidence exists in the backend event model.
