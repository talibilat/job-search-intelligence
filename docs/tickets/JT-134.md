# JT-134 - Breakdown Table And Chart Widgets

## What Changed

The dashboard now renders a Tier 3 breakdown section backed by the deterministic `GET /metrics/breakdown` endpoint.
The section defaults to the source dimension and lets the user switch across role, source, salary, tech, sponsorship, seniority, and work mode.
It shows a Recharts bar chart, a ranked summary, and a table with application, response, interview, and offer counts for each group.
The breakdown endpoint now accepts the shared dashboard filter query fields so grouped metrics compose with the same URL-backed filters as the applications table.
Invalid composed filter ranges return the project-standard typed `422` API error, and role filters escape SQL wildcard characters the same way the applications table does.

## Why It Changed

JT-134 exposes grouped metrics for Phase 3 dashboard segmentation without using any LLM-generated counts.
The displayed values come from the local SQLite `applications` table and `application_events` timeline through the existing metrics API contract.

## Usage Notes

Open `/dashboard` and use the Dimension selector in the Tier 3 segmentation section.
Each selection reloads deterministic grouped rows from `/metrics/breakdown?dimension=<dimension>` plus the active dashboard filter query fields.
The table caption changes with the selected dimension so the grouped values remain accessible to screen-reader and keyboard users.
