# JT-135 - Metric Reconciliation Helpers

## What Changed

Added focused backend reconciliation tests for the deterministic summary and rate metrics APIs.

The tests compare API payload values against the `MetricsRepository` query methods that produce the dashboard numbers.

## Why

JT-135 requires displayed metric values to be traceable back to deterministic local queries.

The reconciliation tests protect the Phase 3 invariant that dashboard facts come from the local SQLite `applications` table and `application_events` timeline, not from an LLM.
