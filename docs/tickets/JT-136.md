# JT-136 - Answer Q-01 Lifetime Applications

## What Changed

JT-136 adds the first deterministic dashboard summary metric for Q-01: lifetime applications.
The backend exposes `GET /metrics/summary` with a typed `MetricsSummaryResponse` containing `total_applications`.
The value is computed with `COUNT(*)` against the canonical local `applications` table.
The dashboard Total applications card now fetches that endpoint through the generated TypeScript API client and renders the reconciled count.

## Why

Q-01 asks how many jobs have been applied to across the user's lifetime job-search history.
The count must be deterministic and must reconcile with the `applications` source-of-truth table.
No LLM is involved in producing the metric.

## How To Use

Run the backend against a migrated local database and request `GET /metrics/summary`.
The response shape is `{"total_applications": <count>}`.
Open the frontend dashboard and read the Total applications card.

## How To Verify

Run `uv run pytest tests/test_metrics_summary_api.py -v` from `backend/`.
Run `npm run test -- src/App.test.tsx` from `frontend/`.
Run the broader backend and frontend verification commands required by AGENTS.md before merging.
