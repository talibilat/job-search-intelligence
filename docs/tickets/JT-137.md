# JT-137 - Answer Q-02 Applications By Window

## What Changed

Added a deterministic `GET /metrics/summary` backend slice for Q-02 application counts.
The endpoint returns week, month, year, and optional custom-window counts from the canonical `applications` table.
The dashboard now renders those counts through the generated OpenAPI client and keeps custom date-window inputs in the route query string.

## Why

Q-02 asks how many applications fall inside this week, this month, this year, or a custom date window.
Counting `applications.first_seen_at` avoids inflating counts from the event timeline, where one application can have many events.
The metric remains deterministic because it uses SQL over local SQLite data and does not involve an LLM.

## How It Works

The backend derives week, month, and year windows from a timezone-aware `anchor_at` query parameter.
Custom windows use `custom_start_at` as an inclusive UTC start and `custom_end_at` as an exclusive UTC end.
All windows are half-open, so an application exactly at one window's end boundary is counted in the next adjacent window instead of both.
