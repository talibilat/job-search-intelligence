# JT-138 - Q-03 Distinct Companies

## What changed

JT-138 adds deterministic support for Q-03, "How many distinct companies have I applied to?"
The backend now exposes `GET /metrics/summary` with a typed `MetricsSummaryResponse` containing `distinct_company_count`.
The dashboard loads that endpoint through the generated TypeScript API client and renders the value as the first metric card.

## Why

Q-03 belongs to Phase 3 deterministic dashboard metrics under FR-3 and FR-3.1.
The count is computed directly from the local SQLite `applications` table and never from an LLM.
Company names are normalized at read time with `LOWER(TRIM(company))` so case and surrounding whitespace do not inflate the distinct-company count.

## Verification

Focused backend tests cover the API response and OpenAPI schema for `/metrics/summary`.
Focused frontend tests cover rendering the Q-03 dashboard card from the generated API client.
Full backend and frontend verification should be run before merging because this change touches both API contract and dashboard UI.
