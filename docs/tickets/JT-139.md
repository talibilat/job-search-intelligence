# JT-139 - Q-04 Response Versus Silence

## What Changed

JT-139 adds a narrow deterministic metrics endpoint for Q-04: `GET /metrics/response-silence`.
The endpoint returns total applications, applications with at least one human response-like timeline event, and applications with total silence.

## Query Contract

The metric reads from the local SQLite `applications` table and its `application_events` timeline.
An application counts as having a human response when it has at least one response-like event type: `response`, `assessment`, `interview_scheduled`, `feedback`, `rejection`, or `offer`.
Applications without any response-like event count as silent, including applications with only `applied` or `ghost_inferred` events.
The same application is counted once even when it has multiple response-like events.

## Why

Q-04 asks how many applications got at least one human response versus total silence.
This metric keeps dashboard truth deterministic and does not involve an LLM.

## Verification

The focused backend tests cover response-like event counting, empty-database zero counts, and OpenAPI documentation for the endpoint.
Run `uv run pytest tests/test_metrics_response_silence_api.py -q` from `backend/` to verify the Q-04 slice.
