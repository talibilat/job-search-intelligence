# JT-140 - Answer Q-05 Rejection Count

## What Changed

JT-140 adds the first deterministic summary metric for Q-05.
`GET /metrics/summary` now returns `rejected_applications`, the count of canonical application rows whose `current_status` is `rejected`.

## Why

Q-05 asks, "How many rejections have I received?"
The ticket defines this as total rejected applications.
The count uses the local SQLite `applications` table as the source of truth and does not involve an LLM.

## How To Verify

Run the focused backend test:

```bash
uv run --project backend pytest backend/tests/test_metrics_summary_api.py
```

For the API contract, regenerate and validate the frontend client after backend verification:

```bash
npm run check
```

Run that command from `frontend/`.
