# JT-141 - Answer Q-06 Ghosted Count

## What Changed

JT-141 adds a deterministic `GET /metrics/summary` backend endpoint for Q-06.
The response includes `ghosted_applications`, `ghost_threshold_days`, and `evaluated_at`.

## Why

Q-06 asks how many applications got no response at all and are therefore ghosted.
The count must use the configured ghost threshold and reconcile with the local `applications` and `application_events` source of truth.

## How It Works

The metrics service computes the silence cutoff from `AppSettings.ghost_threshold_days` at read time.
The repository counts applications that have an applied event, have no response-like event after the latest applied event, and whose latest non-ghost timeline event is older than the configured threshold.
The endpoint is read-only and does not insert or retract `ghost_inferred` events.

## Verification

The focused API test `uv run pytest tests/test_metrics_summary_api.py -q` covers the Q-06 count, configured threshold reporting, and read-only behavior.
