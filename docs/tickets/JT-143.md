# JT-143 - Answer Q-08 Offers Received

## What Changed

JT-143 adds `offers_received` to the deterministic `GET /metrics/summary` response for Q-08.
The dashboard metrics overview now renders an Offers received card from that summary field.

## Why

Q-08 asks how many offers have been received.
The answer must reconcile with the local `application_events` timeline and must not use an LLM-produced count.

## How It Works

The metrics repository counts distinct applications that have at least one `offer` event in `application_events`.
Status-only `applications.current_status = 'offer'` rows do not count unless the timeline contains an offer event.
Multiple offer events for the same application count as one received offer.

## Verification

The focused backend test `uv run pytest tests/test_metrics_summary_api.py -k offers_received -v` covers the event-history count.
The focused frontend test `npm run test -- src/App.test.tsx -t "renders offers received"` covers the dashboard card.
