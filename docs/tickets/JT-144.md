# JT-144 - Q-09 Application Status Table

## What Changed

The dashboard now answers Q-09 by showing a table of applications and their current status.
The table reads from the existing deterministic `GET /applications` API instead of adding a new metrics endpoint or using any LLM-derived data.

## Why

Q-09 asks for the current status of every application.
The existing applications API already exposes typed `ApplicationRecord` rows from the local SQLite `applications` table and supports the filters needed for this dashboard slice.
Rendering that API directly keeps `applications` and `application_events` as the factual source of truth.

## How To Use

Open `/dashboard`.
Use the dashboard filter form to filter by status, role, source, sponsorship, work mode, first-seen date range, or salary band.
Apply filters to update the URL query string and reload the application status table from `/applications` with matching query parameters.
Click an application company name to open its application detail route.
