# JT-146 - Q-11 Overall Response Rate

## What Changed

Added `GET /metrics/rates` for the Q-11 overall response-rate metric.

The response includes `overall_response_rate.numerator`, `overall_response_rate.denominator`, and `overall_response_rate.rate` so the displayed value can be reconciled to deterministic counts.

The dashboard Response rate card calls this endpoint and shows the formatted percentage plus the numerator and denominator evidence text.

The denominator is the total number of rows in the canonical `applications` table.

The numerator is the number of distinct applications with at least one response-like event in `application_events`.

Response-like events use the existing `RESPONSE_LIKE_APPLICATION_EVENT_TYPES` contract and include response, assessment, interview, feedback, rejection, and offer evidence.

Ghost-inferred events do not count as responses.

When there are no applications, the rate is `null` because there is no meaningful denominator.

## Why

JT-146 covers Phase 3, FR-3.2, NFR-5, NFR-7, and Q-11.

The implementation keeps the app's dashboard truth deterministic and grounded in the local SQLite `applications` plus `application_events` source of truth.

No LLM output is used to produce the count, denominator, or rate.
