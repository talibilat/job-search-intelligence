# JT-147 - Q-12 Rejection Rate

## What Changed

Extended `GET /metrics/rates` with a `rejection_rate` metric for Q-12.

The response includes `rejection_rate.numerator`, `rejection_rate.denominator`, and `rejection_rate.rate` so the dashboard value can be reconciled to deterministic counts.

The denominator is the total number of rows in the canonical `applications` table.

The numerator is the number of applications whose canonical `current_status` is `rejected`.

The dashboard now renders a Rejection rate card from the deterministic `/metrics/rates` endpoint alongside the existing response-rate card.

When there are no applications, the rate is `null` because there is no meaningful denominator.

## Why

JT-147 covers Phase 3, FR-3.2, NFR-5, NFR-7, and Q-12.

The implementation keeps the dashboard truth grounded in local SQLite `applications` data and avoids any LLM-derived count or rate.

## Usage Notes

Read Q-12 from `GET /metrics/rates` at `rejection_rate`.

Use the included numerator and denominator text when showing the percentage so users can trace the displayed number back to deterministic application rows.
