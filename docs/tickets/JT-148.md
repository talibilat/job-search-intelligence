# JT-148 - Q-13 Ghost Rate

## What Changed

Extended `GET /metrics/rates` with a `ghost_rate` metric for Q-13.

The response includes `ghost_rate.numerator`, `ghost_rate.denominator`, and `ghost_rate.rate` so the dashboard value can be reconciled to deterministic counts.

The denominator is the total number of rows in the canonical `applications` table.

The numerator is the number of applications that are ghosted or silent past the configured `ghost_threshold_days` cutoff.

The dashboard now renders a Ghost rate card from the deterministic `/metrics/rates` endpoint alongside the existing response and rejection rate cards.

When there are no applications, the rate is `null` because there is no meaningful denominator.

## Why

JT-148 covers Phase 3, FR-3.2, NFR-5, NFR-7, and Q-13.

The implementation keeps the dashboard truth grounded in local SQLite `applications` plus `application_events` data and avoids any LLM-derived count or rate.

## Usage Notes

Read Q-13 from `GET /metrics/rates` at `ghost_rate`.

Use the included numerator and denominator text when showing the percentage so users can trace the displayed number back to deterministic application rows and the configured ghost threshold.
