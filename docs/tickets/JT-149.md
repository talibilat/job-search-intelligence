# JT-149 - Q-14 Application To Interview Rate

## What Changed

Extended `GET /metrics/rates` with an `application_to_interview_rate` metric for Q-14.

The response includes `application_to_interview_rate.numerator`, `application_to_interview_rate.denominator`, and `application_to_interview_rate.rate` so the dashboard value can be reconciled to deterministic counts.

The denominator is the total number of rows in the canonical `applications` table.

The numerator is the number of distinct applications with at least one `interview_scheduled` event in `application_events`.

The dashboard now renders an Application to interview rate card from the deterministic `/metrics/rates` endpoint alongside the existing response, rejection, and ghost rate cards.

When there are no applications, the rate is `null` because there is no meaningful denominator.

## Why

JT-149 covers Phase 3, FR-3.2, NFR-5, NFR-7, and Q-14.

The implementation keeps the dashboard truth grounded in local SQLite `applications` plus `application_events` data and avoids any LLM-derived count or rate.

## Usage Notes

Read Q-14 from `GET /metrics/rates` at `application_to_interview_rate`.

Use the included numerator and denominator text when showing the percentage so users can trace the displayed number back to deterministic application rows and interview timeline events.
