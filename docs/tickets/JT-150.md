# JT-150 - Q-15 Interview To Offer Rate

## What Changed

Extended `GET /metrics/rates` with an `interview_to_offer_rate` metric for Q-15.

The response includes `interview_to_offer_rate.numerator`, `interview_to_offer_rate.denominator`, and `interview_to_offer_rate.rate` so the dashboard value can be reconciled to deterministic counts.

The denominator is the number of distinct applications with at least one `interview_scheduled` event in `application_events`.

The numerator is the number of distinct applications with an `offer` event that occurs after an `interview_scheduled` event using the repository's deterministic event ordering.

The dashboard now renders an Interview to offer rate card from the deterministic `/metrics/rates` endpoint alongside the existing response, rejection, ghost, and application-to-interview rate cards.

When there are no interviewed applications, the rate is `null` because there is no meaningful denominator.

## Why

JT-150 covers Phase 3, FR-3.2, NFR-5, NFR-7, and Q-15.

The implementation keeps the dashboard truth grounded in local SQLite `applications` plus `application_events` data and avoids any LLM-derived count or rate.

## Usage Notes

Read Q-15 from `GET /metrics/rates` at `interview_to_offer_rate`.

Use the included numerator and denominator text when showing the percentage so users can trace the displayed number back to deterministic interview and offer timeline events.
