# JT-151 - Q-16 Full Funnel

## What Changed

Q-16 is now answered on the dashboard by the Application funnel section.
The dashboard reads deterministic stage counts from `GET /metrics/funnel` and reloads the funnel with the same URL-backed filters used by the rest of the dashboard.

The backend funnel endpoint returns the approved stages in order: `applied`, `screen`, `interview`, `final`, and `offer`.
The `final` stage remains `0` until final-round evidence exists in the event taxonomy.

The implementation that completed the behavior was merged through the JT-126 and JT-132 slices.
This ticket note records the JT-151 question-level completion and usage contract.

## Why It Changed

JT-151 maps to Phase 3, FR-3.2, NFR-5, NFR-7, and Q-16.
The product requirement is that factual dashboard answers reconcile with local SQLite `applications` and `application_events` data.

The funnel is deterministic and does not use an LLM for counts.

## Usage Notes

Open `/dashboard` and review the Application funnel section.
Apply dashboard filters such as status, source, role, salary band, sponsorship, or work mode to reload `/metrics/funnel` with the same filtered population.

Use `GET /metrics/funnel` directly for API-level verification.
The endpoint accepts the shared metric filter query fields and returns ordered `stages` rows.
