# JT-153 - Q-18 Time To Rejection

## What Changed

Extended `GET /metrics/summary` with an `average_time_to_rejection` metric for Q-18.
The response includes `average_time_to_rejection.application_count` and `average_time_to_rejection.average_hours` so the dashboard can reconcile the displayed duration to deterministic rejection-event rows.

The metric measures from `applications.first_seen_at` to the first `rejection` timeline event on each application.
Applications without rejection evidence are excluded from the average.

The dashboard now renders an Avg time to rejection card from the deterministic `/metrics/summary` endpoint.
When no applications have rejection evidence, `average_hours` is `null` because there is no meaningful average.

## Why

JT-153 covers Phase 3, FR-3.2, NFR-5, NFR-7, and Q-18.
The implementation keeps the dashboard truth grounded in local SQLite `applications` plus `application_events` data and avoids any LLM-derived timing or count.

Use the included application count when showing the duration so users can trace the displayed average back to deterministic rejection-evidence rows.
