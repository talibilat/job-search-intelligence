# JT-155 - Application Volume Trend

## What Changed

The dashboard now renders Q-20, "How has my application volume trended over time?"
It loads deterministic daily application-volume points from `GET /metrics/timeseries` and displays them in an application-volume trend section.
The section includes a Recharts line chart and a text summary of each returned point so the answer remains visible even when chart rendering is constrained.
`GET /metrics/timeseries` now accepts the shared dashboard metric filters, and the dashboard passes its route-backed filters to the Q-20 request.

## Why

Q-20 belongs to FR-3.2 in Phase 3, where dashboard time-series answers must come from deterministic application data.
The existing backend timeseries endpoint already computes application counts from canonical `applications.first_seen_at` values, so this ticket adds the missing user-facing dashboard answer without changing metric math.
Filter support was threaded through the existing metrics filter DTO so Q-20 composes with the rest of the dashboard controls.

## Usage Notes

Start the local backend and open the dashboard.
The "Application volume trend" section answers Q-20 from `/metrics/timeseries`.
If no applications exist yet, the section shows an empty state instead of implying precision that is not backed by local data.
