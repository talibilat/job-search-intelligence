# JT-161 - Salary Band Conversion

## What Changed

Metric breakdown rows now include deterministic `response_rate`, `interview_rate`, and `offer_rate` fields alongside the existing application, response, interview, and offer counts.

The rates are calculated from the local SQLite `applications` table and `application_events` timeline by dividing each grouped count by that group's application count.

The dashboard Tier 3 segmentation section now renders those conversion rates in the ranked summary and table, including the salary dimension used for Q-26.

The generated OpenAPI schema and TypeScript client were regenerated so frontend callers can consume the new typed fields.

## Why It Changed

Q-26 asks what salary bands are being targeted and how those bands convert.

The existing breakdown endpoint already grouped applications by salary band, but it only exposed raw counts.

Adding typed conversion rates makes the salary-band answer explicit and keeps the calculation deterministic instead of leaving the UI or user to infer it.

## Usage Notes

Open `/dashboard`, choose `Salary` in the Tier 3 segmentation dimension selector, and review each salary band's application count plus response, interview, and offer conversion rates.

The same fields are available from `GET /metrics/breakdown?dimension=salary` and compose with the existing dashboard filters.
