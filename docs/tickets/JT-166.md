# JT-166 - Seniority Conversion Breakdown

## What Changed

`GET /metrics/breakdown?dimension=seniority` now groups raw application seniority values into the Q-31 buckets `junior`, `mid`, `senior`, `lead`, and `unknown`.
The breakdown continues to return deterministic application, response, interview, and offer counts from local SQLite data.

## Why It Changed

Q-31 asks which seniority levels convert best.
Raw extraction values such as `Junior Engineer`, `Mid-level`, `Sr. Software Engineer`, and `Principal / Staff Lead` should not fragment the dashboard into one-off labels.
Canonical buckets make conversion rows comparable and keep the answer deterministic.

## Linked Requirements

FR-3, FR-3.3, NFR-5, NFR-7, Phase 3, and Q-31.
