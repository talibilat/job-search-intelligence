# JT-167 - Dashboard Smoke Test

## What Changed

Expanded the existing Playwright smoke test to cover the fixture-backed dashboard path.
The smoke test now mocks setup status, sync status, applications, summary metrics, rates, and source breakdown responses.
It verifies that the dashboard renders deterministic metric cards, the application status table, and the Tier 3 breakdown table from those fixture responses.

## Why It Changed

JT-167 requires a browser-level smoke check for setup, sync status, fixture dashboard load, and metric rendering.
This keeps the critical Phase 3 dashboard path covered without introducing live backend, Gmail, or LLM dependencies into the smoke suite.

## Linked Requirements

FR-3, NFR-5, NFR-7, and Phase 3.

## Verification

- `npm run check`
- `npm run test:smoke`

## Usage Notes

Run `npm run test:smoke` from `frontend/`.
The test uses Playwright route mocks for local-first API responses and does not require a running backend.
