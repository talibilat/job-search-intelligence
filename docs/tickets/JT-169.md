# JT-169 - Diagnostics Endpoint

## What Changed

Added `GET /metrics/diagnostics` to the metrics API.
The route is thin and delegates diagnostic calculations to `DiagnosticsService`.
It supports the shared dashboard metrics filters and returns typed `MetricsDiagnosticsResponse` data.

## Why It Changed

Phase 3.5 diagnostics need an API surface for deterministic segment comparisons.
The endpoint exposes the JT-168 diagnostics service while preserving local SQLite `applications` and `application_events` as the factual source of truth.
It does not use LLMs for quantitative results.

## Linked Requirements

FR-3.4, FR-3.6, Phase 3.5, and Tier 4.

## Usage Notes

Call `GET /metrics/diagnostics` for unfiltered diagnostic comparisons.
Add shared dashboard filter query fields such as `source=linkedin`, `status=rejected`, or `work_mode=remote` to scope the diagnostic population.
