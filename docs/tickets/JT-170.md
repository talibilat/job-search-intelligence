# JT-170 - Diagnostic Comparison Widgets

## What Changed

Added Phase 3.5 diagnostic comparison widgets to the dashboard.
The dashboard now fetches `GET /metrics/diagnostics` with the active route-backed filters.
It renders the baseline response rate, strongest response signals, weakest response signals, and a deterministic response-rate lift summary.

## Why It Changed

JT-170 requires deterministic winners, losers, and correlation summaries on screen.
The widgets use the diagnostics API from JT-169 and keep all displayed numbers grounded in local `applications` and `application_events` data.
No LLM-generated counts or narrative claims are introduced.

## Linked Requirements

FR-3.4, FR-3.6, Phase 3.5, and Tier 4.

## Verification

- `npm run test -- DashboardPage.test.tsx`
- `npm run check`

## Usage Notes

Open the dashboard and review the `Diagnostic comparisons` section.
Dashboard filters compose with the diagnostics endpoint in the same route-backed way as the other metric widgets.
