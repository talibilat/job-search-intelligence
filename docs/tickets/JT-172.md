## JT-172 - Answer Q-32 common traits of successful applications

## What changed

`GET /metrics/diagnostics` now includes deterministic successful-application diagnostics for Q-32.
Successful applications are applications with interview or offer evidence in `application_events`.

The diagnostics response now returns:

- `baseline_success_count`
- `baseline_success_rate`
- `successful_application_segments`
- `success_count`, `success_rate`, and `success_rate_lift` on diagnostic segment comparisons

The dashboard diagnostics card now renders a `Q-32 successful application traits` section that shows the strongest overrepresented traits among interview or offer applications.

## Why it changed

Q-32 asks what successful applications have in common.
The existing diagnostics foundation compared response-rate lift, but it did not isolate interview or offer outcomes as the success cohort.

The new logic keeps the answer deterministic by deriving success counts from local SQLite `applications` and `application_events` rows.
It does not use an LLM for counts, rates, or comparisons.

## Usage notes

Call `GET /metrics/diagnostics` with the existing dashboard metric filters to retrieve Q-32 diagnostics for the filtered population.
Read `successful_application_segments` to identify traits whose interview or offer success rate is above the filtered baseline.

Open the dashboard and review the `Q-32 successful application traits` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
