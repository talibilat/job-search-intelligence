## JT-173 - Answer Q-33 common traits of rejected or ghosted applications

## What changed

`GET /metrics/diagnostics` now includes deterministic negative-outcome diagnostics for Q-33.
Negative outcomes are applications whose canonical `current_status` is `rejected` or `ghosted`.

The diagnostics response now returns:

- `baseline_negative_count`
- `baseline_negative_rate`
- `negative_outcome_segments`
- `negative_count`, `negative_rate`, and `negative_rate_lift` on diagnostic segment comparisons

The dashboard diagnostics card now renders a `Q-33 rejected or ghosted traits` section that shows the most overrepresented traits among rejected or ghosted applications.

## Why it changed

Q-33 asks what rejected or ghosted applications have in common.
The existing diagnostics response highlighted weakest response-rate segments, but did not isolate rejected or ghosted outcomes as their own cohort.

The new logic keeps the answer deterministic by deriving negative outcomes from local SQLite `applications.current_status` values and comparing each segment against the filtered baseline.
It does not use an LLM for counts, rates, or comparisons.

## Usage notes

Call `GET /metrics/diagnostics` with the existing dashboard metric filters to retrieve Q-33 diagnostics for the filtered population.
Read `negative_outcome_segments` to identify traits whose rejected-or-ghosted rate is above the filtered baseline.

Open the dashboard and review the `Q-33 rejected or ghosted traits` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
