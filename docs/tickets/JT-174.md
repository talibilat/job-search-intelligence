## JT-174 - Answer Q-34 strongest response correlate

## What changed

`GET /metrics/diagnostics` now includes `strongest_response_correlate` for Q-34.
The value is the single segment with the strongest positive response-rate lift above the filtered baseline.

The dashboard diagnostics card now renders a `Q-34 strongest response correlate` section that names the factor and describes it as the strongest positive correlate.

## Why it changed

Q-34 asks which single factor correlates most with getting a response.
The diagnostics endpoint already ranked strongest response segments, but it did not expose the top segment as an explicit Q-34 answer.

The implementation keeps the answer deterministic by reusing the local SQLite segment comparisons over `applications` and `application_events`.
It does not use an LLM for counts, rates, or ranking.

## Usage notes

Call `GET /metrics/diagnostics` with the existing dashboard metric filters and read `strongest_response_correlate` for the Q-34 answer.
If no segment has positive lift, the field is `null`.

Open the dashboard and review the `Q-34 strongest response correlate` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
