## JT-175 - Answer Q-35 wasted-effort segments

## What changed

`GET /metrics/diagnostics` now includes `wasted_effort_segments` for Q-35.
These are the segments with negative response-rate lift relative to the filtered baseline.

The dashboard diagnostics card now renders a `Q-35 wasted-effort segments` section that names below-baseline segments and shows their deterministic response-rate lift.

## Why it changed

Q-35 asks whether effort is going into role or company-type segments that never convert.
The diagnostics endpoint already computed weakest response segments, but it did not expose them as an explicit wasted-effort answer.

The implementation keeps the answer deterministic by reusing local SQLite segment comparisons over `applications` and `application_events`.
It does not use an LLM for counts, rates, or ranking.

## Usage notes

Call `GET /metrics/diagnostics` with the existing dashboard metric filters and read `wasted_effort_segments` for the Q-35 answer.
An empty list means no segment is currently below the filtered response baseline.

Open the dashboard and review the `Q-35 wasted-effort segments` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
