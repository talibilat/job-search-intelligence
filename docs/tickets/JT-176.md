## JT-176 - Answer Q-36 best ROI source

## What changed

`GET /metrics/diagnostics` now includes `best_roi_source` for Q-36.
The value is the source segment with the highest interview rate, using interviews per application as the ROI measure.

The dashboard diagnostics card now renders a `Q-36 best ROI source` section that names the best source by interview ROI.

## Why it changed

Q-36 asks which application source gives the best ROI.
The existing metrics could show source breakdowns, but the diagnostics response did not expose one explicit best source answer.

The implementation keeps the answer deterministic by deriving interview rates from local SQLite `applications` and `application_events` data.
It does not use an LLM for counts, rates, or ranking.

## Usage notes

Call `GET /metrics/diagnostics` with the existing dashboard metric filters and read `best_roi_source` for the Q-36 answer.
If no source has interview evidence, the field is `null`.

Open the dashboard and review the `Q-36 best ROI source` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
