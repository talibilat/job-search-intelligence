## JT-177 - Answer Q-37 sponsorship response impact

## What changed

`GET /metrics/diagnostics` now includes `sponsorship_response_impact` for Q-37.
The value is the sponsorship segment with the lowest response-rate lift relative to the filtered baseline.

The dashboard diagnostics card now renders a `Q-37 sponsorship response impact` section that quantifies the response-rate difference in percentage points.

## Why it changed

Q-37 asks whether sponsorship requirements are measurably hurting response rate, and by how much.
The existing sponsorship breakdowns showed segment rates, but the diagnostics response did not expose one explicit sponsorship impact answer.

The implementation keeps the answer deterministic by deriving sponsorship response lift from local SQLite `applications` and `application_events` data.
It does not use an LLM for counts, rates, or comparisons.

## Usage notes

Call `GET /metrics/diagnostics` with the existing dashboard metric filters and read `sponsorship_response_impact` for the Q-37 answer.
If no sponsorship segment is available, the field is `null`.

Open the dashboard and review the `Q-37 sponsorship response impact` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
