## JT-178 - Answer Q-38 selling versus dead-weight skills

## What changed

`GET /metrics/diagnostics` now includes deterministic skill signal diagnostics for Q-38.

The diagnostics response now returns:

- `selling_skill_segments`
- `dead_weight_skill_segments`

The dashboard diagnostics card now renders a `Q-38 selling vs dead-weight skills` section that highlights skills with interview evidence and, when available, below-baseline skill segments.

## Why it changed

Q-38 asks which listed skills correlate with interviews versus which are dead weight.
The existing tech breakdowns could show conversion by skill, but diagnostics did not expose a dedicated skill signal answer.

The implementation keeps the answer deterministic by deriving interview and response conversion from local SQLite `applications` and `application_events` data.
It does not use an LLM for counts, rates, or ranking.

Call `GET /metrics/diagnostics` with the existing dashboard metric filters and read `selling_skill_segments` plus `dead_weight_skill_segments` for the Q-38 answer.

Open the dashboard and review the `Q-38 selling vs dead-weight skills` section inside `Diagnostic comparisons` to see the same deterministic answer on screen.
