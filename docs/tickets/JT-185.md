# JT-185 - Insight Grounding Validator

## Summary

JT-185 adds a service-level grounding validator for generated narrative insights.
The validator rejects provider output before caching when it does not cite at least one citation ID from the deterministic insight input evidence.
It also rejects citation-like IDs that were not present in the input payload.
It rejects factual claim text that is not covered by a valid citation in the same sentence or the immediately following citation-only sentence.

## Why

FR-4 requires Tier 5 insights to cite the applications or emails they are drawn from.
This keeps cached narrative insights traceable to the local `applications` table and its event timeline.
It also prevents ungrounded provider output from becoming trusted cached content.

## Behavior

`InsightGenerationService` still builds deterministic facts and evidence before calling the configured LLM provider.
After the provider returns text, the service validates finish reason, non-empty content, and grounding citations.
Only content whose claims cite provided `citation_id` values is persisted to the `insights` cache.
Bracketed prose that does not look like a citation remains allowed, but bracketed citation-like tokens must match the prepared input evidence.
Ungrounded content raises the existing public-safe `LLMProviderResponseError` path and leaves the cache unchanged.

## Verification

Focused insight tests cover accepted grounded output, cache reuse, forced regeneration, invalid provider output, and ungrounded provider output.
Run `uv run --project backend pytest backend/tests/test_insight_generation_service.py backend/tests/test_insight_input_builder.py backend/tests/test_insight_repository.py -q` from the repository root.
