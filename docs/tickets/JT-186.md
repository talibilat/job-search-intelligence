# JT-186 - Add Insights API Routes

## What Changed

JT-186 adds backend API routes for Phase 4 cached insights.
`GET /insights` returns the latest cached insight records from local SQLite, including stale records so the UI can tell the user when regeneration is needed.
`POST /insights/regenerate` accepts an insight type and forces that insight through the existing generation service, LLM provider seam, deterministic input builder, and repository cache write.

## Why

FR-4 requires cached narrative insights for Tier 5 questions.
FR-4.2 specifically requires cached loading plus user-triggered regeneration for cost control.
The routes keep API handlers thin while the service layer owns read and generation behavior.

## API Usage

List cached insights:

```http
GET /insights
```

Regenerate one insight:

```http
POST /insights/regenerate
Content-Type: application/json

{
  "type": "why_rejected"
}
```

The regenerate request also accepts `max_evidence_items`, defaulting to `100`, for the existing generation service evidence cap.
