# JT-188 - Add Insight Cost Display

## What Changed

`POST /insights/regenerate` now returns a typed `cost` payload with estimated prompt tokens, estimated completion tokens, estimated USD cost, actual provider token usage, and actual USD cost when provider usage and pricing are available.
The insight generation service estimates the prompt from the exact provider request payload and uses the configured maximum output token budget for completion estimates.
The `/insights` page now stores the most recent regeneration cost per insight type and displays estimated cost, actual cost, and actual token count after a successful regeneration.
The backend environment example now documents the non-secret insight pricing settings.

## Why It Changed

JT-188 maps to FR-4 and NFR-2.
Regeneration is intentionally user-triggered for cost control, and the user needs visibility into estimated and actual spend when an insight is regenerated.
The change keeps cost accounting at the API boundary and does not make LLM-generated content authoritative for dashboard counts.

## Usage Notes

Configure non-local insight pricing with `JOBTRACKER_INSIGHT_INPUT_COST_PER_1K_UNITS_USD` and `JOBTRACKER_INSIGHT_OUTPUT_COST_PER_1K_UNITS_USD` when pricing is known.
Leave those settings at `0` when pricing is unknown; the API still returns token estimates and marks cost estimates unavailable.
Ollama local mode reports zero estimated and actual cost.
After an insight is regenerated in the UI, the card shows estimated cost, actual cost, and actual provider token count when the response includes usage.
