# JT-189 - Q-40 Recurring Rejection Themes

## What Changed

JT-189 narrows the `why_rejected` insight to Q-40: why rejections happen and what recurring themes appear across rejection emails.
The insight generation prompt now gives explicit Q-40 instructions for `why_rejected` outputs.
The Q-40 evidence input now uses rejection events only, so recruiter feedback summaries do not leak into the recurring rejection theme answer.
The insight input hash now includes the insight prompt version, so the Q-40 prompt change bypasses older cached `why_rejected` rows.

## Why

Q-40 asks for recurring themes across rejection emails.
Feedback-specific summaries belong to neighboring Tier 5 work, especially Q-41.
Keeping Q-40 scoped to rejection events reduces cross-ticket overlap and keeps the generated narrative tied to the source emails that actually represent rejections.
