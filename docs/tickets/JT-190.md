# JT-190 - Answer Q-41 Recurring Recruiter Feedback

## What Changed

JT-190 adds `recurring_feedback` as the cached insight type for Q-41.
The insight generation input builder scopes this question to `feedback` timeline events only, so the LLM receives cited recruiter or interviewer feedback evidence instead of generic rejection evidence.
The existing insight generation service can now generate, cache, and retrieve this type through the same grounded LLM path used by the other Phase 4 insight types.

## Why

Q-41 asks what recruiter or interviewer feedback consistently says the user should improve.
That is narrower than Q-40 rejection themes, so it needs a separate insight type and evidence scope.
The implementation preserves the `applications` plus `application_events` source-of-truth invariant and keeps deterministic facts separate from narrative synthesis.
