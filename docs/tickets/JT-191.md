# JT-191 - Q-42 Rejected-Role Skill Gaps

JT-191 implements the Phase 4 Q-42 insight behavior for skill and technology gaps in rejected roles.
The insight input builder now prepares a deterministic `rejected_skill_counts` fact from rejected applications only.
Those counts come from the local `applications.tech_stack` field and are scoped to applications whose `current_status` is `rejected`.

The `skill_gaps` evidence package is scoped to rejected applications so interview, offer, and other winning signals are not treated as gaps by default.
The LLM generation prompt adds Q-42-specific instructions telling the provider to identify recurring technologies and skills in rejected roles, use only deterministic facts and cited evidence, and avoid treating wins as gaps unless rejected-role evidence also supports the claim.

This keeps the LLM in the Phase 4 narrative role.
It may synthesize the answer, but the rejected-role skill counts and citations are prepared deterministically before generation.

Verification for this ticket should include the insight input builder tests, the insight generation service tests, backend ruff, backend mypy, and relevant backend pytest coverage.
