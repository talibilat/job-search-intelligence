# JT-192 - Q-43 Strongest And Weakest Signals

## Summary

JT-192 adds the `strongest_weakest_signals` cached insight type for Q-43.
The insight uses the existing Phase 4 insight generation service and cache path.
Its input builder uses whole-history application and event evidence so the LLM can compare positive and negative patterns without making unsupported counts.

## Scope

The change updates the typed insight model, the SQLite insight type constraint, and the insight input evidence scope.
It does not add API routes or frontend rendering because those are owned by separate insights API and UI tickets.
It does not change aggregation, classification, provider adapters, redaction, or shared docs.

## Grounding Behavior

The generation prompt continues to require the LLM to use only deterministic facts and cited evidence.
The Q-43 input includes all application evidence, not just rejection or open-application evidence.
Cached `strongest_weakest_signals` rows are invalidated by the same application, event, and raw-email staleness triggers as the other cached insights.
