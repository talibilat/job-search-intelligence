# JT-193 - Answer Q-44 Best-Fit Roles

## What Changed

JT-193 adds deterministic role outcome summaries to the `role_fit` insight input.
The summaries group included cited applications by role title and count interview or offer outcomes as wins, rejected or ghosted outcomes as losses, and per-status totals as cited facts for synthesis.
Each role summary carries the citation IDs used to ground the role-fit claim.
The `role_fit` prompt now explicitly answers Q-44 by asking the LLM to identify which roles genuinely suit the user best from patterns of wins.
The `role_fit` evidence scope stays unfiltered so Q-44 can compare wins, losses, and thin open-role evidence from the same cited source package.

## Why

Q-44 is a Tier 5 cached narrative insight under FR-4.1.
The LLM may synthesize the narrative only after deterministic facts and cited application or email evidence are prepared.
Providing `role_outcome_summaries` keeps the role-level win and loss pattern outside the model's inference path while still requiring citations for every role-fit claim.
The implementation keeps API routes, frontend surfaces, schema, migrations, provider adapters, classification, and aggregation untouched because sibling tickets own those areas.
