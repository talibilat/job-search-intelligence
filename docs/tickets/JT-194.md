# JT-194 - Answer Q-45 Next-Week Actions

## What Changed

JT-194 tightens the `weekly_actions` insight generation path for Q-45.
The generated insight must now answer the question, "What are the 3 concrete things I should do next week to improve outcomes?"
The service prompt asks the configured LLM provider for exactly three numbered actions, one per line, with each action grounded in provided evidence.

## Grounding And Validation

The `weekly_actions` output is validated before it is cached.
It must contain exactly three non-empty numbered lines, numbered `1.` through `3.`.
Each line must include at least one citation that matches a `citation_id` from the deterministic `InsightInput` evidence payload.
If the provider returns a blank, incomplete, uncited, citation-only, misnumbered, wrong-count, or unknown-citation response, the service rejects it instead of saving an ungrounded recommendation.

## Cache Behavior

The insight input hash now includes `INSIGHT_GENERATION_PROMPT_VERSION`.
Prompt-template changes therefore bypass legacy cached rows even when the underlying deterministic facts and evidence are unchanged.

## Scope

This change stays inside the existing Phase 4 cached-insights service seam.
It does not add a new API route, migration, provider adapter, frontend page behavior, telemetry, shared credentials, auto-apply behavior, or outbound email behavior.

## Verification
