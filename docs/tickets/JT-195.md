# JT-195 - Answer Q-46 Search Story

## What Changed

Q-46 now uses the existing `story` insight type to generate the job-search narrative.
The insight input builder narrows story evidence to the most recent 12-month window anchored to the newest cited evidence.
The resulting evidence remains chronological so the LLM receives a source-grounded search arc instead of an unordered pile of events.
The insight generation prompt now names Q-46 directly and asks for a cited narrative of the last 6 to 12 months.
Insight input hashes now include the insight generation prompt version so prompt changes invalidate cached narratives.

## Why

Q-46 asks what story the last 6 to 12 months of the search tells.
The existing `story` type was only generic cached-insight plumbing.
It did not scope evidence to the recent search period or instruct the provider to write the specific Q-46 narrative.

## How To Verify

Run the focused backend tests from `backend/`:

```sh
uv run pytest tests/test_insight_input_builder.py::test_story_input_uses_recent_chronological_search_window tests/test_insight_generation_service.py::test_insight_generation_service_uses_q46_story_prompt -q
```

Run the relevant insight test set from `backend/`:

```sh
uv run pytest tests/test_insight_input_builder.py tests/test_insight_generation_service.py tests/test_insight_repository.py -q
```

Run backend linting and type checks from `backend/`:

```sh
uv run ruff check app/services/insights_service.py tests/test_insight_input_builder.py tests/test_insight_generation_service.py
uv run mypy
```
