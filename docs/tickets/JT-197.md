# JT-197 - Add Email Chunking Service

## What changed

Added a backend email chunking service that builds deterministic, unembedded text chunks from retained job-related email bodies.
Added typed DTOs for chunk source rows and unembedded text chunks.
Added an `EmailRepository` query that only returns retained, non-empty raw email bodies with a stored job-related classification.

## Why it changed

Phase 5 RAG chat needs a safe chunking seam before embeddings and sqlite-vec storage are added by later tickets.
This keeps JT-197 limited to deterministic text preparation and avoids crossing into JT-198 embedding provider work or JT-199 vector repository work.

## Verification

The focused JT-197 test was run with:

```bash
uv run pytest tests/test_email_chunking_service.py -v
```

## Usage notes

Use `EmailChunkingService.build_chunks(limit=...)` with an `EmailRepository` to produce `EmailTextChunk` DTOs.
The service only reads emails returned by `EmailRepository.list_chunkable_retained_emails`, which excludes metadata-only rows, debugging bodies, blank retained bodies, and rows not classified as job-related.
Embedding generation and sqlite-vec persistence are intentionally left to later Phase 5 tickets.
