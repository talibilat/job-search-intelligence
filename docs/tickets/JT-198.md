# JT-198 - Add Embedding Provider Interface

## What changed

Extended the provider-neutral `LLMProvider` strategy with an `embed` method.
Added typed embedding request, vector, and response DTOs with 1536-dimensional vector validation for the existing sqlite-vec schema.
Added Azure OpenAI and Ollama embedding request paths behind the existing provider adapters.
Added focused tests for the provider contract and both adapters.

## Why it changed

Phase 5 RAG chat needs a provider seam for embedding retained job-related chunks before later tickets generate and persist embeddings.
This keeps embedding calls behind the same LLM provider strategy boundary as chat generation, preserving provider swapability and public-safe error handling.

## Verification

Focused provider tests were run with:

```bash
uv run pytest tests/test_llm_provider_contract.py tests/test_azure_openai_provider.py tests/test_ollama_llm_provider.py -v
```

## Usage notes

Call `LLMProvider.embed(LLMEmbeddingRequest(...))` with retained local text chunks to receive `LLMEmbeddingResponse` values.
The DTOs hide retained text and embedding values from repr output.
Chunk selection, sqlite-vec persistence, and semantic retrieval remain owned by later Phase 5 tickets.
