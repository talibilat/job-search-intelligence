# JT-202 - Add Structured Query Tool

## What changed

Added a Phase 5 structured-query tool under `app.agent.tools`.
The tool accepts only typed, whitelisted templates for total applications, summary counts, rates, funnel, and breakdown metrics.
It delegates all metric reads to `MetricsRepository` and exposes grounded deterministic rows for later chat synthesis.

## Why it changed

The RAG chat agent needs a quantitative tool that can answer counts, rates, and breakdowns without letting an LLM generate SQL.
This preserves dashboard consistency by reusing the same deterministic metrics repository instead of adding a separate query path.

## Verification

Focused tests were run with:

```bash
uv run pytest tests/test_structured_query_tool.py -v
```

## Usage notes

Call `StructuredQueryTool.run(StructuredQueryRequest(...))` with one of the whitelisted templates.
`StructuredQueryRequest` forbids extra fields and has no raw SQL field.
Chat routing, synthesis, semantic retrieval, and API streaming remain later Phase 5 slices.
