# JT-206 - Add Chat History Endpoint

## What changed

Added `GET /chat/history` backed by SQLite chat history.
Added a `ChatHistoryService`, `ChatHistoryResponse`, and a `ChatRepository.list_messages` method.
The endpoint supports optional `conversation_id` filtering and a bounded `limit` parameter.

## Why it changed

Phase 5 chat needs persisted local chat history so prior conversations can be viewed without introducing remote storage.
The endpoint reads only existing compact chat rows, citations, and tool outputs from SQLite.

## Verification

Focused tests were run with:

```bash
uv run pytest tests/test_chat_history_api.py -v
```

## Usage notes

Call `GET /chat/history` to list local chat messages.
Use `conversation_id` to narrow results to one conversation.
Use `limit` to bound the returned messages; the API enforces a range of 1 to 500.
