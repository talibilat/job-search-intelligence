# JT-230 - Build grounded chat backend and retrieval pipeline

## Context

The app currently has a disabled chat drawer and a read-only history endpoint.
Phase 5 requires a grounded hybrid RAG agent that can answer quantitative questions through deterministic tools and content questions through cited retrieval.
The LLM must never emit raw SQL for execution.

## Linked Requirements

- FR-5.1 hybrid routing.
- FR-5.2 structured-query tool.
- FR-5.3 semantic retrieval tool.
- FR-5.5 grounded answers.
- FR-5.6 persisted chat history.
- FR-6.1 local-first storage.
- NFR-8 security.
- Q-47 through Q-50.

## Scope In

- Implement email chunk persistence for retained job-related bodies using sqlite-vec.
- Generate embeddings through the configured provider path and keep embeddings local in SQLite.
- Implement semantic search over retained job-related chunks with citations to emails and applications.
- Wire the existing structured-query tool into a constrained quantitative path.
- Implement hybrid routing for quantitative, content, and mixed questions.
- Implement grounded synthesis that cites tool outputs and refuses unsupported claims.
- Add `POST /chat` with streaming or a clear incremental response contract.
- Persist user, assistant, tool, and citation data in SQLite.
- Add backend tests for routing, structured-query correctness, semantic retrieval, grounding, persistence, and privacy constraints.

## Scope Out

- Do not add arbitrary SQL execution.
- Do not answer with uncited private-email claims.
- Do not send metadata-only unrelated emails to embeddings.
- Do not build draft-writing or autonomous outbound email.

## Technical Notes

- Use LangGraph if it remains the selected implementation path.
- Quantitative answers must reconcile exactly with deterministic metric endpoints.
- Content answers must cite real emails or applications.
- Store compact citations and tool outputs in `chat_messages`.
- Keep LLM provider behavior behind provider adapters.

## Acceptance Criteria

- [ ] `POST /chat` can answer at least one quantitative question using deterministic metrics.
- [ ] `POST /chat` can answer at least one content question using semantic retrieval and citations.
- [ ] Mixed questions use both structured-query and semantic retrieval paths.
- [ ] Chat answers cite real source emails, applications, or deterministic metric outputs.
- [ ] No raw SQL from an LLM is executed.
- [ ] Chat history is persisted and retrievable through `GET /chat/history`.
- [ ] Wipe-data removes chat history and embeddings.
- [ ] Backend tests cover Q-47 through Q-50 representative cases.
