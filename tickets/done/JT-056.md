Implementation notes:
- Added `GET /auth/gmail/callback` under the auth router.
- Kept the route thin by delegating state validation, provider callback completion, and connection persistence to the Gmail auth service.
- Passed OAuth callback codes through a `SecretStr` DTO boundary.
- Exchanged authorization codes inside the Gmail provider and rejected non-`gmail.readonly` scopes.
- Fetched the Gmail profile email address to scope the account connection.
- Stored access and refresh token payloads through the configured `SecretStore` and persisted only non-secret `email_connections` metadata.
- Returned `EmailConnection` metadata without raw authorization codes, client secrets, access tokens, or refresh tokens.
- Unified keyring and Fernet secret-store factory paths for token persistence.
- Regenerated the frontend OpenAPI schema and Orval API client for the new callback endpoint.
