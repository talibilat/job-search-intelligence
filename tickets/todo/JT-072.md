This ticket contributes to the local-first job-search intelligence app by preserving the `applications` table and event timeline as the factual source of truth.

- Follow Repository pattern for data access, Strategy for email and LLM providers, Pipeline stages passing Pydantic DTOs, service layer business logic, FastAPI dependency injection, typed errors, mypy-clean code, and ruff formatting.

- Ensure secrets are encrypted at rest and never logged.
- The LLM never emits raw SQL for execution; use deterministic SQL, typed Python logic, or constrained query builders as appropriate.
- Reuse the `EmailSyncStatus` DTO and `GET /sync/status` response added by JT-073.
- Planned files/modules: backend/app/api/, backend/app/providers/email/, backend/app/services/sync_service.py, docs/, tickets/.
