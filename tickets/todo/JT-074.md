Title: JT-074 - Connect sync API to persisted Gmail account

Mini-PRD / Context:
Connect the existing `POST /sync` and `GET /sync/status` routes to persisted Gmail connection state once OAuth callback and token persistence land.
JT-073 added the route shell, manual sync runtime seam, typed not-configured response, concurrency response, and status DTO; this ticket now covers replacing the temporary unconfigured dependency with real connected-account lookup.
This ticket contributes to the local-first job-search intelligence app by preserving the `applications` table and event timeline as the factual source of truth.

Linked requirements:
FR-1, FR-6, NFR-1, NFR-4, NFR-7, Phase 1

Scope - in:
- Resolve the configured Gmail account for the current local user without exposing token material.
- Inject an `EmailConnection` into the existing sync runtime when Gmail connection persistence is available.
- Preserve the existing typed `400` response when no Gmail account is connected.
- Keep the change atomic enough for one focused implementation session.
- Touch the planned modules only unless implementation discovery shows a documented need.

Scope - out:

- Do not introduce telemetry, shared credentials, autonomous outbound email, auto-apply behavior, or multi-user SaaS assumptions.
- Do not add broad abstractions outside the architecture contract.
- Do not move later-phase work into this ticket unless explicitly listed.

Technical approach:

- Follow Repository pattern for data access, Strategy for email and LLM providers, Pipeline stages passing Pydantic DTOs, service layer business logic, FastAPI dependency injection, typed errors, mypy-clean code, and ruff formatting.
- Keep API routes thin and put business behavior in services.
- Ensure secrets are encrypted at rest and never logged.
- The LLM never emits raw SQL for execution; use deterministic SQL, typed Python logic, or constrained query builders as appropriate.
- Reuse the existing `POST /sync` and `GET /sync/status` route handlers from JT-073 instead of adding duplicate routes.
